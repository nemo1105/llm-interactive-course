# 第一章页面发布计划

## 当前发布判断

当前版本可以进入本地人工走查和小范围内部复审；不应作为最终演讲版本或公开分享版本发布。发布前必须解决 fullPage.js 授权、licenseKey 或署名策略，并完成人工内容、视觉和演示节奏复审。

## 发布步骤

| 步骤 | 负责人 | 前置条件 | 成功信号 | 中止条件 |
| --- | --- | --- | --- | --- |
| 运行本地验证 | human | 依赖已安装，工作区没有未理解的冲突改动。 | `pnpm run verify:iteration` 通过。 | 任一自动化检查失败。 |
| 人工浏览首页和第一章 | human | 自动化验证通过。 | 首页、章节首页、两个演示页和下一章预留页均可按预期浏览。 | 页面出现未确认内容、错位、遮挡或演示节奏不可接受。 |
| 复核授权和署名 | human | 已确认 release readiness 中的 fullPage.js 风险。 | 明确采用购买授权、展示署名或替换实现方案之一。 | licenseKey 警告或署名要求未解决。 |
| 确认演讲版本 | human | 人工内容、视觉、节奏和授权复审均通过。 | 记录 go 决策和发布窗口。 | 任一发布阻断项仍存在。 |

## 就绪证据

- 成功路径覆盖：`pnpm run verify:iteration` 于 2026-06-24 通过，包含文档检查、类型检查、13 条单元测试和 11 条 Playwright 测试。
- 失败路径覆盖：自动化覆盖禁用占位表达、未来消息提前出现、首页边界破坏、中性预留页、payload 格式边界和 step 引用完整性。
- 边界或异常覆盖：自动化覆盖短视口 payload、左右浮层定位、长工具调用时序、内部滚动、顶部工具栏稳定和 JSON 折叠展开。
- 补偿验证：人工复审负责判断中文表达、视觉密度、演示节奏和 fullPage.js 授权或署名。
