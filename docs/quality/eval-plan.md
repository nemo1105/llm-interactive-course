# 第一章页面评估计划

## 评估行为

- 学习者能看到直接文本路径和工具调用路径的结构差异。
- 章节首页能说明本章要讲什么，并提供进入演示和下一章导航。
- 演示页只保留单一顶部工具栏，并能通过其中的步进控制逐步展示用户可见聊天状态和第一章传输事件。
- 左侧聊天状态和右侧标准时序图进度保持同步；右侧当前事件对应的左侧消息块保持可见并高亮，聊天内容未溢出时从顶部自然排列，溢出时当前消息靠近聊天区域底部。
- 第一章事件 payload 能在消息标签 hover/focus 时展示完整传输数据，未来 payload 不提前出现，浮层能根据空间选择右、左、下、上任一合理位置并保持可读。
- 页面使用第一章已确认示例，不使用第一章禁用占位表达。
- 首页占位状态不被章节实现破坏。

## 通过标准

- `pnpm run verify:iteration` 通过。
- Playwright 能验证章节首页、两个演示页、下一章预留页、顶部演示选择、顶部工具栏稳定、步进同步、左侧聊天固定与聚焦消息、右侧时序图内部滚动、标准时序图、payload hover 查看入口、浮层内 API 格式切换、左右侧智能定位和短视口浮层边界。
- 内容数据测试能确认两个演示、上海天气 mock 数据、完整 payload、step 序列和引用完整性存在，并阻止第一章禁用占位表达进入页面。

## 人工复审

- 自动化通过后，人工复审视觉密度、中文表达和演示节奏。
- 人工复审未完成前，不把页面标记为最终演讲版本。
