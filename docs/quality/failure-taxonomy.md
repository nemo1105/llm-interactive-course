# 第一章页面失败分类

## 类型

- 第一章内容 gate 失败：页面仍使用第一章禁用占位表达、未确认真实服务数据，或暗示 mock 数据来自实时服务。
- 机制表达失败：页面只展示聊天效果，没有展示第一章传输事件、请求、响应或工具结果回写。
- 步进同步失败：演示不可步进、未来消息线提前出现、左侧聊天状态和右侧时序图进度不一致，或右侧当前事件没有明确对应并高亮左侧消息块。
- 图渲染失败：标准时序图区域空白、参与者或消息线不可见，或退回自由节点图。
- 交互状态失败：章节导航、演示路由、顶部演示选择、payload hover/focus 查看、浮层内 API 格式切换或下一章预留路由错误。
- payload 可读性失败：JSON payload 未使用可折叠树、默认展开策略掩盖教学主干字段，或辅助字段无法按需展开查看。
- 回归失败：首页占位、文档检查、类型检查或自动化测试被破坏。
- 视觉可用性失败：布局拥挤、文字重叠、移动端难以阅读，右侧内部滚动导致页面级滚动或左侧聊天内容离开主体区域，payload 浮层底部内容在短视口中被截断，或浮层没有利用左右空闲空间导致遮挡关键消息线或顶部工具栏。

## 严重度

- 阻断：内容 gate、路由不可访问、自动化验证失败。
- 高：核心交互路径错误或机制表达缺失。
- 中：视觉密度、文案节奏或非关键辅助区问题。
- 低：不影响理解的小样式问题。
