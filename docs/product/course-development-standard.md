# 课程开发事实记录

## 文档边界

本文件只记录课程产品事实、已确认取舍和需要下游章节引用的本地背景；规则来源见 `AGENTS.md`。

## 已确认产品事实

- 课程从一次普通 LLM 对话开始，再逐步揭示多轮历史、消息 role、工具调用、token、推理参数、输出形态、前缀缓存、提示词、上下文工程、最小 agent loop、MCP、RAG、memory、skills、错误处理、运行环境、文件、上下文压缩、计划任务、通道、连接器、权限审批、人机协作、Agent 协作、观测和评估。
- 第 12 章是后续 Agent 工程章节的基础准则：Agent 并不神秘，下一步由模型基于当前可见上下文输出，工程层负责执行、约束、拦截、路由、授权和停止。
- MCP 放在 RAG 前讲，用于说明工具、资源和 prompts 的标准化连接；RAG 基础同时讲工程外部检索插入和模型通过检索工具自行生成检索词/句两种方式。
- Memory 必须讲两种实现：工程侧像 RAG 一样检索后插入记忆信息；以及系统/开发者提示词要求模型使用记忆工具完成检索、写入、更新和删除。
- Skills 是课程核心章节之一，重点讲技能目录、入口文件、绝对路径、文件读取加载、渐进式暴露、文档/脚本/二进制/模板组合和多技能协作。
- 运行环境既可以由 developer prompt 固定提供，也可以由工具查询；多环境切换通过环境切换工具完成，普通业务工具不需要都暴露 environment id 参数。
- 文件章节区分可直接进入模型上下文的图片类文件，以及以元数据和路径形式暴露、由工具脚本操作的办公文件和源码文件。
- 上下文超限章节强调模型上下文从 32K 到 1M 仍会不足，压缩必然丢信息，应让模型结合任务场景总结压缩历史并保留来源。
- 计划任务章节讲定时任务系统，不讲 agent 自己写 todo 执行计划；到点执行仍是一次新的模型调用。
- 通道章节讲飞书、钉钉、WhatsApp、自定义 API 等消息通道；连接器章节讲三方系统授权、OAuth、授权发现和平台 CLI 工具。
- 人机协作用“环”表达 AI 执行弧线、人类保留关键决策点；Agent 协作用多个环形成“流”，核心是工程层提供通讯工具、Agent 发现工具和工作流程描述。
- 权限审批章节必须明确高风险动作拦截发生在工程层，工具执行前停止并展示确认弹窗。
- 第一受众是具备基础 AI 或编程概念、但未必构建过 LLM 系统的混合技术听众。
- 章节示例应尽量复用少量连续语境，优先沿用上海客户现场出行、A 客户续约风险、B公司 2026年SLA合同、客户会议纪要、邮箱、日历、CRM、文件和通道等语境。
- 示例实体命名应使用简称或中性代号，例如 A 客户、B公司 2026年SLA合同、张江客户现场；既保留具体对象，又避免像真实人名或真实公司名。
